📋 0. 事前準備・ヒアリング

店舗オーナーから事前に取得する情報です。

🗄 1. Supabase プロジェクト作成

店舗専用のSupabaseプロジェクトを作成します。

1
Supabase Dashboard を開き、「New Project」 をクリック。
2
以下を入力：
・Name: akinai-店舗名-英語（例：akinai-bar-sakura）
・Database Password: 生成して 必ず控える
・Region: Northeast Asia (Tokyo)
3
作成完了後、Settings → API から以下を控える：
・Project URL（例：https://xxxxxx.supabase.co）
・anon public key
・service_role key（秘密鍵。外部に公開しない）
🔒 service_role key は絶対に外部公開しないこと。Edge Functionの環境変数としてのみ使用する。

🏗 2. テーブル・初期データ作成

SQL Editorで一括実行します。

1
Supabase Dashboard → SQL Editor を開く。
2
付録A：SQLテンプレート のSQLを 全文コピーして貼り付け、実行する。
3
Table Editorでテーブルが作成されていることを確認：
menu_items orders customers config prompts broadcast_log ai_usage_log
4
config テーブルを開き、以下の値を店舗に合わせて編集：
・SHOP_NAME → 店舗名
・SHOP_ICON → 絵文字
・ADMIN_PASSWORD → 管理画面パスワード
💡 SQLテンプレートにはデフォルトの config 値と prompts が含まれているので、実行するだけで初期データも入ります。

⚡ 3. Edge Function デプロイ

APIサーバーとなるEdge Functionをデプロイします。

1
テンプレートの Edge Function ソースを用意。
場所：C:\Users\6kimu\akinai-ai-template\supabase\functions\api\index.ts
2
ターミナルで Supabase CLI にログイン： 
npx supabase login
npx supabase link --project-ref 【Project ID】
3
環境変数を設定： 
npx supabase secrets set SUPABASE_URL=【Project URL】
npx supabase secrets set SERVICE_ROLE_KEY=【service_role key】
npx supabase secrets set GEMINI_API_KEY=【Gemini API Key】
npx supabase secrets set CHANNEL_ACCESS_TOKEN=【LINE Channel Access Token】
npx supabase secrets set LIFF_ID=【LIFF ID】
4
デプロイ： 
npx supabase functions deploy api --no-verify-jwt
5
デプロイ後のURL（https://xxxxxx.supabase.co/functions/v1/api）を控える。これが GAS_API_URL になる。
⚠️ --no-verify-jwt を忘れるとフロントエンドからのリクエストが401エラーになる。

📦 4. Storage バケット作成

メニュー画像・ロゴ画像の保存先を作成します。

1
Supabase Dashboard → Storage → 「New Bucket」
2
バケット名：menu-images、Public bucket を ON にする。
3
バケット内に items フォルダを作成（メニュー画像格納先）。
📌 RLS（Row Level Security）はPublicバケットなので設定不要。画像URLは https://xxxxxx.supabase.co/storage/v1/object/public/menu-images/items/ファイル名 の形式になる。

💚 5. LINE公式アカウント作成

店舗専用のLINE公式アカウントを作成します。

1
LINE for Business から新規アカウントを作成。
2
アカウント名は 店舗名 を設定。
3
プロフィール画像に店舗ロゴを設定。
4
LINE Official Account Manager → 設定 → 応答設定：
・応答モード：Bot
・Webhook：ON
・自動応答メッセージ：OFF
・あいさつメッセージ：任意（カスタマイズ推奨）
💡 既存のLINE公式アカウントがあればそれを利用してもOK。応答設定だけ変更する。

🔧 6. LINE Developers 設定

Messaging APIチャネルとLIFFアプリを作成します。

6-1. Messaging API チャネル

1
LINE Developers → プロバイダー選択 → 「チャネル作成」→「Messaging API」
2
チャネル名：店舗名、チャネル説明：適当に入力。
3
「Messaging API」タブ → Channel access token を発行して控える。
4
Webhook URL を設定： 
https://【Supabase Project URL】/functions/v1/api
Webhookの利用を ON にする。

6-2. LIFFアプリ作成

5
同じチャネル内の 「LIFF」タブ → 「追加」
6
設定値：
・Size：Full
・Endpoint URL：https://app.【店舗ドメイン】（後で設定するカスタムドメイン）
・Scope：profile にチェック
・Bot link feature：Aggressive
7
作成後に表示される LIFF ID を控える。
🔒 Channel access token はEdge Functionの環境変数としてのみ使用。コードにハードコードしない。

📂 7. GitHub リポジトリ作成・デプロイ

テンプレートから店舗用リポジトリを作成し、GitHub Pagesでデプロイします。

1
テンプレートリポジトリ akinai-ai-template を 「Use this template」→「Create a new repository」 で複製。
リポジトリ名：akinai-【店舗名英語】（例：akinai-bar-sakura）
2
script.js の以下2箇所を書き換え： 
var GAS_API_URL = "https://【Supabase URL】/functions/v1/api";
var MY_LIFF_ID = "【LIFF ID】";
3
admin.html の API URL も同様に書き換え： 
var GAS_API_URL = "https://【Supabase URL】/functions/v1/api";
4
config.js が存在する場合、同様に URL を書き換え。
5
GitHub → Settings → Pages → Source: Deploy from a branch、Branch: main / root を選択 → Save。
6
数分後に https://【username】.github.io/akinai-【店舗名】/ でアクセス可能になる。
💡 テンプレートリポジトリを事前に作成しておくと、毎回の作業は「複製 → URL2箇所書き換え → push」だけで済む。

🌐 8. DNS・カスタムドメイン設定

店舗専用のサブドメインを設定します。

1
Squarespace DNS（またはDNS管理画面）を開く。
2
CNAME レコードを追加： 
Host: app.【店舗名】
Type: CNAME
Value: 【GitHubユーザー名】.github.io
3
GitHub → リポジトリの Settings → Pages → Custom domain に app.【店舗名】.allbrain.one を入力。
4
Enforce HTTPS にチェック（DNS反映後に有効化される。最大30分程度）。
5
LINE Developers → LIFF の Endpoint URL を https://app.【店舗名】.allbrain.one に更新。
⚠️ DNS反映には最大24時間かかる場合がある（通常は数分〜30分）。HTTPS有効化はDNS反映後にしかできない。

✅ 9. 接続テスト・動作確認

全体が正しく連携しているか確認します。

🚨 動作確認が全て完了するまで店舗への引き渡しは行わないこと。

🍽 10. メニュー登録・初期設定

ヒアリング内容をもとにメニューと各種設定を登録します。

1
管理画面 → メニュー管理 から、ヒアリングしたメニューを1つずつ登録。
・メニューID規則：ドリンク=D001〜、フード=F001〜
・味覚パラメータは必ず設定する（0のままにしない）
2
設定タブ で以下を確認・調整：
・ショップ名 / アイコン / ロゴ
・テーマモード
・VIP閾値（来店回数・期間）
・AIプロンプト（店舗の雰囲気に合わせてカスタマイズ）
3
AI利用状況タブ → 自動配信を ON にし、配信時間を設定。
4
店舗オーナーに以下を納品：
・管理画面URL
・管理画面パスワード
・操作マニュアルURL（/manual.html）
・LINE公式アカウントのQRコード
・卓上POP用QRコード画像
💡 メニュー数が多い場合は、CSVからのバルクインポートSQLを使うと効率的（付録A参照）。

📎 付録A：SQLテンプレート

Supabase SQL Editorにコピペして実行するSQL全文です。

-- ===========================
-- アキナイAI テーブル作成SQL
-- ===========================

-- 1. menu_items
CREATE TABLE IF NOT EXISTS menu_items (
  id SERIAL PRIMARY KEY,
  item_id TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  price INTEGER NOT NULL DEFAULT 0,
  category TEXT NOT NULL DEFAULT 'Drink',
  flavor_profile TEXT DEFAULT '',
  pairing_food TEXT DEFAULT '',
  is_active BOOLEAN DEFAULT true,
  is_soldout BOOLEAN DEFAULT false,
  image_url TEXT DEFAULT '',
  p_salty INTEGER DEFAULT 0,
  p_sweet INTEGER DEFAULT 0,
  p_sour INTEGER DEFAULT 0,
  p_bitter INTEGER DEFAULT 0,
  p_rich INTEGER DEFAULT 0,
  options TEXT DEFAULT '',
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. orders
CREATE TABLE IF NOT EXISTS orders (
  id SERIAL PRIMARY KEY,
  order_id TEXT NOT NULL,
  line_user_id TEXT NOT NULL,
  user_name TEXT DEFAULT '',
  item_id TEXT NOT NULL,
  item_name TEXT NOT NULL,
  price INTEGER DEFAULT 0,
  quantity INTEGER DEFAULT 1,
  options TEXT DEFAULT '',
  status TEXT DEFAULT 'pending',
  table_id TEXT DEFAULT '',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ,
  paid BOOLEAN DEFAULT false
);

-- 3. customers
CREATE TABLE IF NOT EXISTS customers (
  id SERIAL PRIMARY KEY,
  line_user_id TEXT UNIQUE NOT NULL,
  display_name TEXT DEFAULT '',
  picture_url TEXT DEFAULT '',
  visit_count INTEGER DEFAULT 0,
  last_visit TIMESTAMPTZ,
  first_visit TIMESTAMPTZ DEFAULT NOW(),
  total_spent INTEGER DEFAULT 0,
  tags TEXT DEFAULT '',
  memo TEXT DEFAULT '',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 4. config
CREATE TABLE IF NOT EXISTS config (
  id SERIAL PRIMARY KEY,
  key TEXT UNIQUE NOT NULL,
  value TEXT DEFAULT ''
);

-- 5. prompts
CREATE TABLE IF NOT EXISTS prompts (
  id SERIAL PRIMARY KEY,
  key TEXT UNIQUE NOT NULL,
  value TEXT DEFAULT ''
);

-- 6. broadcast_log
CREATE TABLE IF NOT EXISTS broadcast_log (
  id SERIAL PRIMARY KEY,
  segment TEXT DEFAULT 'all',
  message TEXT DEFAULT '',
  sent_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 7. ai_usage_log
CREATE TABLE IF NOT EXISTS ai_usage_log (
  id SERIAL PRIMARY KEY,
  action TEXT NOT NULL,
  user_id TEXT DEFAULT '',
  tokens_used INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
