Step 0 – 事前ヒアリング情報の準備

新規店舗を立ち上げる前に、以下の情報をクライアントから取得してください。

項目例
店舗名Bar さくら
店舗アイコン（絵文字）🍷
ロゴ画像logo.png（2 MB 以下）
メニュー一覧Excel or スプレッドシート
LINE 公式アカウント既存 or 新規開設
希望ドメインapp.bar-sakura.allbrain.one
管理画面パスワード任意の文字列
テーマdark / light

Step 1 – Supabase プロジェクト作成

  1. Supabase Dashboard にログインし「New project」をクリック
  2. プロジェクト名を入力（例：akinai-bar-sakura）
  3. データベースパスワードを設定し、安全に保管
  4. Region は Northeast Asia (Tokyo) を選択
  5. 作成完了後、以下を記録：
    • Project URL：https://xxxxxxxx.supabase.co
    • anon public key：Settings → API → anon / public
    • service_role key：Settings → API → service_role（秘密鍵・絶対に外部に出さない）

Supabase Dashboard → SQL Editor で以下の SQL を実行してください。全文は 付録 A にもまとめています。

Step 4 – Edge Function デプロイ

ソースコードは supabase/functions/api/index.ts にあります。

4-1. Supabase CLI ログイン & リンク

npx supabase login
npx supabase link --project-ref <Project-ID>

4-2. 環境変数の設定

npx supabase secrets set SUPABASE_URL=<Project URL>
npx supabase secrets set SUPABASE_SERVICE_ROLE_KEY=<service_role キー>
npx supabase secrets set GEMINI_API_KEY=<Google Gemini API キー>
npx supabase secrets set CHANNEL_ACCESS_TOKEN=<LINE Channel Access Token>
npx supabase secrets set LIFF_ID=<LIFF アプリ ID>
npx supabase secrets set STRIPE_SECRET_KEY=<Stripe 秘密鍵>
npx supabase secrets set STRIPE_WEBHOOK_SECRET=<Stripe Webhook シークレット>
🔒 SUPABASE_SERVICE_ROLE_KEY・STRIPE_SECRET_KEY は絶対にフロントエンドやリポジトリに含めないでください。

4-3. デプロイ

npx supabase functions deploy api --no-verify-jwt

デプロイ完了後の URL をメモ：

https://<project-ref>.supabase.co/functions/v1/api

この URL を以後 GAS_API_URL として使用します。

Step 4.5 – 自動配信スケジューラ（pg_cron）

AI 自動配信（runAutoBroadcast）を毎日自動実行するために、Supabase の pg_cron を設定します。

4.5-1. Extension の有効化

Supabase Dashboard → Database → Extensions で以下を有効化：

  • pg_cron
  • pg_net

4.5-2. cron ジョブの登録

SQL Editor で以下を実行：

SELECT cron.schedule(
  'auto-broadcast-daily',
  '0 1 * * *',   -- 毎日 UTC 01:00 = JST 10:00

  $$
  SELECT net.http_post(
    url := 'https://<project-ref>.supabase.co/functions/v1/api',
    headers := '{"Content-Type":"application/json"}'::jsonb,
    body := '{"action":"runAutoBroadcast"}'::jsonb
  );

  $$
);

4.5-3. 登録確認

SELECT * FROM cron.job;

ジョブが表示されれば設定完了です。

💡 テスト用に即時実行したい場合は、SQL Editor で直接 SELECT net.http_post(…) を実行してください。

Step 5 – Storage バケット作成

Supabase Dashboard → Storage で以下の2つのバケットを作成します。

5-1. menu-images

  1. 「New bucket」をクリック
  2. 名前：menu-images
  3. Public bucket を ON
  4. 作成後、バケット内に items フォルダを作成

画像の公開 URL 形式：

https://<project-ref>.supabase.co/storage/v1/object/public/menu-images/items/<filename>

5-2. share-images

  1. 「New bucket」をクリック
  2. 名前：share-images
  3. Public bucket を ON

味覚診断のシェア画像が保存されます。

Step 6 – LINE 公式アカウント設定

  1. LINE Official Account Manager でアカウントを作成（既存があればスキップ）
  2. 応答設定：
    • 応答メッセージ：オフ
    • Webhook：オン
  3. あいさつメッセージを設定（任意）
  4. アカウント名・アイコン・プロフィールを整備

Step 7 – LINE Developers 設定

7-1. Messaging API チャネル

  1. LINE Developers → 該当プロバイダー → Messaging API チャネル
  2. Channel Access Token を発行（長期）しメモ
  3. Webhook URL に Edge Function URL を設定： 
    https://<project-ref>.supabase.co/functions/v1/api
  4. 「Webhook の利用」を ON
  5. 「検証」ボタンで成功を確認

7-2. LIFF アプリ

  1. LINE Developers → 該当チャネル → LIFF タブ → 「追加」
  2. 設定値：
    項目値
    SizeFull
    Endpoint URLhttps://app.<store-domain>
    Scopeprofile
    Bot link featureAggressive
  3. LIFF ID をメモ（1234567890-xxxxxxxx の形式）

Step 8 – GitHub リポジトリ & Pages

8-1. リポジトリ作成

  1. akinai-ai-template を Fork して akinai-<store-en> にリネーム
  2. クローン： 
    git clone https://github.com/allbrain-app/akinai-<store-en>.git

8-2. フロントエンド変数の設定

以下のファイル内の変数を実際の値に置換：

script.js

const GAS_API_URL = 'https://<project-ref>.supabase.co/functions/v1/api';
const MY_LIFF_ID  = '<LIFF ID>';

admin.html（または config.js）

const GAS_API_URL = 'https://<project-ref>.supabase.co/functions/v1/api';
const MY_LIFF_ID  = '<LIFF ID>';

8-3. GitHub Pages 有効化

  1. リポジトリ Settings → Pages
  2. Source：Deploy from a branch
  3. Branch：main / / (root)
  4. Save

公開 URL：https://allbrain-app.github.io/akinai-<store-en>/

Step 9 – カスタムドメイン / DNS

  1. DNS 管理画面で CNAME レコードを追加： 
    app.<store-name>.allbrain.one  →  allbrain-app.github.io
  2. GitHub Pages Settings → Custom domain に app.<store-name>.allbrain.one を入力
  3. Enforce HTTPS を ON
  4. LIFF の Endpoint URL を https://app.<store-name>.allbrain.one に更新
💡 DNS 反映に最大 24 時間かかる場合があります。HTTPS 証明書は GitHub が自動発行します。

Step 10 – 動作確認チェックリスト

#確認項目確認方法
1管理画面ログインhttps://app.<domain>/admin.html でパスワード入力
2メニュー表示注文画面にメニュー一覧が表示される
3LINE 友だち追加LIFF URL を開いて友だち追加フローが動作
4注文フロー商品選択 → カート → 注文確定 → 管理画面に表示
5AI ソムリエ注文画面で「AIおすすめ」が表示される
6顧客一覧管理画面の顧客タブにユーザーが表示
7売上ダッシュボード管理画面の売上分析タブにデータが表示
8ブロードキャストテスト配信を自分宛に送信して LINE で受信確認
9設定変更店舗名・アイコン変更後「全て保存」→ 反映確認
10Webhook 検証LINE Developers の「検証」ボタンで成功

Step 11 – メニュー登録

管理画面の「メニュー管理」タブから登録します。

  1. メニュー ID の命名規則：
    • ドリンク：D001, D002, …
    • フード：F001, F002, …
  2. メニュー名・価格・カテゴリを入力
  3. 画像をアップロード（JPEG/PNG、2 MB 以下）
  4. 味覚パラメータ（塩味・甘味・酸味・苦味・旨味）を 0〜100 で設定
  5. 保存
⚠ 味覚パラメータが未設定のメニューは AI ソムリエの推薦対象外になります。全メニューに必ず設定してください。
